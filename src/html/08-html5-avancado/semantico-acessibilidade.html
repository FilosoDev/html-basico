Introdução ao HTML5 Semântico

15 de janeiro de 2024

O HTML5 introduziu elementos semânticos que dão significado ao conteúdo, melhorando a acessibilidade e SEO. Estes elementos substituem as tradicionais <div> genéricas por tags com propósito específico.

Importante: Elementos semânticos não apenas organizam o código, mas também comunicam estrutura e significado para navegadores, leitores de tela e motores de busca.

Principais Elementos Semânticos

Elementos de Estrutura

<header>
Define o cabeçalho de uma página ou seção
<nav>
Contém links de navegação principais
<main>
Conteúdo principal único da página
<section>
Agrupa conteúdo relacionado tematicamente
<article>
Conteúdo independente que pode ser distribuído
<aside>
Conteúdo relacionado mas não essencial
<footer>
Rodapé de página ou seção

Elementos de Conteúdo

  • <figure> e <figcaption> - Imagens com legendas
  • <time> - Datas e horários
  • <mark> - Texto destacado
  • <details> e <summary> - Conteúdo expansível
Clique para ver exemplo de código HTML5 
<article>
    <header>
        <h2>Título do Artigo</h2>
        <time datetime="2024-01-15">15 de janeiro de 2024</time>
    </header>
    <p>Conteúdo do artigo...</p>
</article>

Acessibilidade Web (WCAG)

Atributos ARIA (Accessible Rich Internet Applications)

ARIA fornece informações semânticas adicionais para tecnologias assistivas:

Principais Atributos ARIA
Atributo Função Exemplo
aria-label Fornece nome acessível aria-label="Fechar modal"
aria-labelledby Referencia elemento que rotula aria-labelledby="title-id"
aria-describedby Referencia descrição adicional aria-describedby="help-text"
aria-expanded Estado de elemento expansível aria-expanded="false"
aria-hidden Oculta elemento de leitores de tela aria-hidden="true"

Marcos de Navegação (Landmarks)

Marcos ajudam usuários de leitores de tela a navegar rapidamente:

  • role="banner" - Cabeçalho principal
  • role="navigation" - Área de navegação
  • role="main" - Conteúdo principal
  • role="complementary" - Conteúdo relacionado
  • role="contentinfo" - Informações da página (rodapé)

Boas Práticas de Desenvolvimento

Checklist de Acessibilidade

  1. Estrutura semântica: Use elementos HTML5 apropriados
    • header, nav, main, section, article, aside, footer
    • h1-h6 em hierarquia lógica
  2. Texto alternativo: Todas as imagens devem ter alt
    • alt="" para imagens decorativas
    • Descrição concisa para imagens informativas
  3. Navegação por teclado: Todos os elementos interativos acessíveis
    • Ordem lógica de tabulação
    • Indicadores visuais de foco
    • Skip links para conteúdo principal
  4. Contraste de cores: Mínimo 4.5:1 para texto normal
    • 7:1 para texto pequeno (melhor prática)
    • 3:1 para textos grandes (18pt+ ou 14pt+ negrito)
  5. Formulários acessíveis:
    • Labels associados a todos os inputs
    • Mensagens de erro claras
    • Instruções e validação em tempo real

Ferramentas de Teste

Ferramentas Recomendadas:

  • axe DevTools: Extensão para análise automática
  • WAVE: Web Accessibility Evaluation Tool
  • Lighthouse: Auditoria built-in no Chrome
  • Screen readers: NVDA (Windows), VoiceOver (Mac)
